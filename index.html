Github Pages使用基于Let’s Encrypt的HTTPs安全链接

Posted on 2016-03-24   |   In tech   |     |   visitors

概述

本文主要针对搭建在Github上的博客，借助Kloudsec平台，可以使在github上的博客使用由Let’s Encrypt Authority X1签发的证书。也就是说，在访问在github上搭建的博客时也可以使用https了~

啥是Let’s Encrypt?(搬运维基)

  • Let’s Encrypt 是一个将于2015年末推出的数字证书认证机构，将通过旨在消除当前手动创建和安装证书的复杂过程的自动化流程，为安全网站提供免费的SSL/TLS证书。
  • Let’s Encrypt 是由互联网安全研究小组（ISRG，一个公益组织）提供的服务。主要赞助商包括电子前哨基金会，Mozilla基金会，Akamai以及思科。2015年4月9日，ISRG与Linux基金会宣布合作。
  • 用以实现这一新的数字证书认证机构的协议被称为自动证书管理环境（ACME）。GitHub上有这一规范的草案，且提案的一个版本已作为一个Internet草案发布。
  • Let’s Encrypt 宣称这一过程将十分简单、自动化并且免费。
    现在Let’s Encrypt已然进入public beta阶段（2016年3月），用户可以较自由的使用Let’s Encrypt签发证书。

使用Kloudsec之后有什么优劣

优势
  1. 快，无需编码，免费！
  2. 无痛地生成https链接
  3. 不用进后台，不用开终端，做genkey之类的事情
  4. 还有CDN的增值服务
劣势
  1. 可能会拖慢站点的访问速度（尤其在中国）
  2. 还在测试中，服务稳定性有待测试

步骤

  1. 进入Kloudsec
  2. 注册（邮箱、密码）、配置Github页面、重写域名的A记录和TXT记录（这一步要去你的DNS服务商更改，如DNSPod）
  3. 点击继续，稍等一会就好了。

总结

这玩意算是个神器吧，直接给网站加了https，他说他是基于Let’s Encrypt（加密后的链接确实是基于Let’s Encrypt的），但是博主无需跟Let’s Encrypt大邑点交道就可以完成https加密。有一点比较致命的缺点，就是你的域名直接指向了他给出的服务器，这貌似就相当你你添加了个你“信任”的中间人，中间人又为你做了CDN和HTTPs，这一步跳转增加了站点响应的时间，也增大了网站内容被篡改、监控的风险。
