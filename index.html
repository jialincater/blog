做了MIDI半年多了，也该为这个库做点贡献了，毕竟这是一个少有人用的库，冷不丁一用还真有点难。和硬件交互没有啥简单的事，都是摸着石头过河。摸摸更健康。

由于这篇文章是中文的，同时也免不了对一些英文博客的引用，因此引用会写在最后，用双语标出。

话说，在iOS MIDI界一定要提的就是Gene De Lisa大神，人家码出来的代码就是能用，虽然，有时候不太工整吧。在我的这段历程中，少不了参考这位大神的，虽然总是没什么用，但是多少会给我一点启发。

最简单的MIDI包发送实现

感觉对于初学者来讲，上苹果官网花半天看了一遍document，看完之后感觉比看之前还糊涂。其实先看看成品代码也没什么不好的

是不是So Easy? 执行完这段代码，你就可以给你的MIDI设备发一个指令了。代码中p1,p2,p3是MIDI包中的数据，规则详见MIDI标准.
当然接收MIDI消息，也可以用类似的机制。另外，在发送的接收消息的时候，推荐使用多线程，因为在发送，接收的同时，你可能还会要求iOS系统播放，或者显示这些消息，一旦时间花销很大，可能会影响整个音乐的节奏。

简单解释一下上面的代码吧~

声明并初始化client和port,这个是发送指令之前必须要干的，在发送之前要明确给谁发，从哪里发。实际上，这段代码千篇一律，基本在所有使用CoreMIDI的代码中都需要。

之后，读过developer文档的同学应该都知道，MIDI指令的data要藏在MIDIPacket里面,MIDIPacket要藏在MIDIPacketList里面，装的方法就是用MIDIPacketListAdd方法。一个MIDIPacketList里面可以有多个MIDIpacket，这样就可以做到向外面发送多个连续的包。在MIDIPacketListAdd方法中，值得一提的是第三个参数，表示packet的生效时间，在上面的例子中传的是0，表示NOW（即时生效），也可以增加其他的MIDI时间量。

最后MIDISend在for循环中，向所有MIDI设备都发一遍MIDIPacketList,此时，在你的MIDI设备上便会有响应了。当然，如果你能确定在众多的MIDI Destination中哪一个是你真正的target，也可以直接进行send.

At last, CoreMIDI库中当然不仅仅有上述这几个方法，但是上面提到的可以说是一定会被用到。有关其他关于MIDI有趣的Point，我会以后再补充。

这个博客写了两年了吧，看了看还真没啥有用的东西，还是坚持写下去吧，至少给自己留一点记忆。

前天弄了个shuttle.cat的域名，刚开始买了半天没买到，人家说支付宝到账要check3天，我太着急了，cancel了好几遍订单。最后被划走了3次钱，现在都没找回来呢。最后还是光光帮忙把他买下来了。

然后这个域名和以往的域名不一样，他有两条特殊规定：

  1. You must develop activities (in any language) that promote the Catalan language and culture
  2. You will publish a website on the internet, within 6 months following Your .CAT domain name registration, which must be fully or partly in Catalan language.

比较硬性的规定就是要用加泰罗尼亚语弄个网页。然后呢，简单wiki之后就可以发现，这个加泰罗尼亚语大概就是巴萨罗那那块说的话，大概和西班牙语差不多吧。

因此，我又马上向我的墨西哥小伙伴问了问，他告诉了我这些：

  • Well I don’t know Catalan, but is like a mixture of Spanish with France and Portuguese
  • A bit explanation is that the real name of the Spanish is “castellano” is another region from Spain but since when they conquer America the king of Spain was from that region they used to speak it and then became the Spanish language and Catalan is from “Cataluña” other region.
  • Even today if you go to there they will speak in Catalan

综上。这个cat语还是和西语不一样的，所以翻译的时候还是按照cat语翻吧~ 然后就是一段故事，大概就是说cat语和西语同样历史悠久，而且沿袭至今。

Setup Mac/iOS VPN configuation using XML & PHP

Veröffentlicht am 2015-09-02

Q: How to set up VPN in a quick way?
A: Use a config file.

In Mac/iOS, there’s a build in config file developed by Apple - *.mobileconfig, here is the description on fileinfo.

Configuration file created for iPhone and iPod touch devices; contains device setup information, such as password policies, access permissions, and proxy connection information, as well as VPN, email, and calendar settings; commonly used for deploying and registering Apple device configurations within enterprise networks.

This kind of configuation file is commonly used in fast config of muti-devices. In addition, if you are providing some service you can also use this kind of file.

Generally speaking, building such a file need an Apple developed software called Apple Configurator, and you can enter your information into this software and than export a “.mobileconfig” file.

This is a good way, however, this is not a intellgent way. Because for most users, setting up VPN config is a little bit complex and time consuming. What they really need is a UI that only need you to enter their account name and their password. That would be perfect.

So you should do something behind this beautiful interface. You should know the structure of the .mobileconfig file, and figure out where to save the username ans where to save the password.So I figured it out. It is actually a XML file using DTD provided by apple. In this XML file, your configuration is saved.

<?xml version=\”1.0\” encoding=\”UTF-8\”?>
<!DOCTYPE plist PUBLIC \”-//Apple//DTD PLIST 1.0//EN\” \”http://www.apple.com/DTDs/PropertyList-1.0.dtd\">

With all these information supported, it’s not hard for us to build a php file which receive the HTTP POST request from the front end and generate such a XML file on your server. This XML file contains the unique user info and your server info. If you want to change your server, just correct the XML and let users login to re-download the mobileconfig file.

The last but not the least, you still need to sign your mobileconig file to make it safe and relible. To sign your config file, use this:

openssl smime -sign -signer path/to/crt -inkey path/to/key -certfile path/to/ca-bundle -nodetach -outform der -in name.mobileconfig -out profile_signed.mobileconfig

Everything will be alright then.

Learning Baysian Network

Veröffentlicht am 2015-05-18

Outline

In this project, a BN and a DBN is learned from the given data using Greedy Hill Climbing and random restarts. A directed graph without any edge is seen as the initial graph, then an algorithm called GHC is used and with this method, a better structure is found step by step supported by the scoring function, until a situation that every neighbor of the current status have lower score then the current, then random restart is executed. In this way, the best structure will be get. After that, parameter of this structure can be calculate so that inference can be done to predict the value.

Basic Steps

BN StructureLearning

At the very beginning, structure learning is necessary to find out the best-scored structure in BN.
With the initial network which has N vertex generated by the data class, in every step in GHC, program will find out the better scored structure in its neighbor, that structure will be seem as current structure. In this way, the current structure will be better and better until the program find a structure that all its neighbor have lower score, that means, a local best is found. To deal with this situation, random restart will be executed, a random graph generator will be called and a new random graph will be returned as the current structure, then GHC is applied again. After several times of random restart, a global best is found.
In the progress above, a TABU list is also applied when checking the neighbor of the current structure, if the neighbor is in the TABU list, ignore it and check the next one, else, calculate its score and add it into TABU list.

DBN StructureLearning

Analogously, structure learning in DBN also use GHC algorithm, random restart and TABU lists. The difference is that structure learning in GHC has 2N vertex(Which is double number of BN), but has no edge from time t to time t or for time t+1 to time t. When learning DBN structures, if a vertex has 3 parent vertex, which is the max number of a vertex, there will not any additional edge points to it.

Efficiency

Array, ArrayList and LinkedList

The time complexity comparison is as follows:


add() in the table refers to add(E e), and remove() refers to remove(int index)
ArrayList has O(n) time complexity for arbitrary indices of add/remove, but O(1) for the operation at the end of the list.
LinkedList has O(n) time complexity for arbitrary indices of add/remove, but O(1) for operations at end/beginning of the List.
The difference of their performance is obvious.

LinkedList is faster in add and remove, but slower in get. Based on the complexity table and testing results, we can figure out when to use ArrayList or LinkedList. In brief, LinkedList should be preferred if:
there are no large number of random access of element
there are a large number of add/remove operations
In our project, the data class need only add data once(The data never changes), but is read several times, so ArrayList is a good choice for data. For other small data structures, LinkedList is seem in high efficiency, and sometimes for the small length-fixed structures, array is used.

### Directed Graph(Adjacency matrix or adjacency list)
Adjacency matrix or adjacency list? When facing this problem, it is important to make a decision for that to continue our project. It can not only have impact on the efficiency of the program but also can influence the stability. Multiple factor is considered in our project, as listed below.

As can be seen in the chart above. An adjacency matrix occupy much more space in the memory, and adjacency list just save what it need. However, adjacency matrix have a higher speed to find a specific edge, but it is slower to iterate though the graph.
In our project, it is rare to have a graph with more than 20 vertex, so it will not occupy lots of memory. What’s more, we always need to look up parents of a specific vertex, which need to Iterate though the graph, so a data structure using adjacency list is generated.

Data Structure in Parameter Learning

As we known, the score is typically based on the count[ ][ ][ ] and the counts[ ][ ] in this project. However, iterating the whole data is time-consuming. So we need to estimate the number of different values in each of the dimensions when they are initialized.
Although the number of count[ ][ ][ ] and counts[ ][ ] is fixed given the structure, it is impossible to determine how many parent configurations or even which nodes are the parents of a certain node. So in order not to receive exceptions in the project. We initial the count[ ][ ][ ] and count[ ][ ] with maximum value for secure purpose. Also it turns out to save the memories comparing to the condition if we initialize them with a relatively large number.

Polymorphic

Polymorphic in scoring function

The scoring function is where we implement the polymorphism to redefine the method resultOfScore based on different kinds of scoring criterion. Concerning that there are various kinds of algorithm to score and select the best structure independently. We provide the base class scoringfunction where all the attributes are declared in this class.
Also, we made the scoringfunction class an abstract class with an abstract method resultOfScore, so you can’t not initialize an object of scoringfunction type, instead the specific type is recommended at the beginning.
